# Simulação de Agricultura de Precisão Tolerante a Falhas com Drones e CRDTs Delta

Este projeto de **Trabalho de Conclusão de Curso (TCC)** visa simular um cenário de *agricultura de precisão* tolerante a falhas, usando *Conflict-Free Replicated Data Types* (**CRDTs**) em uma rede de drones. Em ambientes de **Internet das Coisas (IoT)** e **computação de borda**, manter a consistência dos dados entre dispositivos distribuídos (como drones e sensores agrícolas) é desafiador devido a conexões intermitentes e limitações de hardware. Técnicas tradicionais de replicação de dados costumam exigir sincronização cara ou consenso, o que é impraticável em dispositivos com recursos limitados . CRDTs evitam consenso, usando regras matemáticas que garantem convergência eventual mesmo com atualizações concorrentes . Isso os torna adequados para sistemas distribuídos tolerantes a falhas, pois asseguram **Consistência Eventual Forte** – todos os nós convergem para o mesmo estado se receberem os mesmos updates, mesmo que fora de ordem . Em particular, **Delta State CRDTs** (δ-CRDTs) enviam apenas deltas (mudanças incrementais) em vez do estado inteiro, combinando benefícios dos CRDTs baseados em estado e em operação . Assim, δ-CRDTs reduzem o tráfego de rede, importante para drones que se comunicam por links sem fio instáveis.

No projeto, duas alunas dividirão tarefas complementares (Parte A e Parte B) que convergem em um sistema integrado. A **Parte A** focará no desenvolvimento do núcleo de dados: um **Delta-Set CRDT** implementado em Go e incorporado em “**drones gateways**” simulados (contêineres Docker que trocam deltas entre si). A **Parte B** construirá a infraestrutura de simulação: orquestração dos múltiplos nós em Docker Compose, coleta de métricas (desempenho, convergência, uso de banda etc.) e injeção de falhas de rede controladas. Ao final, ambas irão analisar os resultados e **redigir um artigo técnico** para conferência acadêmica (nas áreas de redes tolerantes a falhas, IoT, CRDTs ou edge computing), demonstrando a viabilidade da abordagem. A seguir, apresentamos um cronograma detalhado de **6 meses**, com divisão clara de responsabilidades, marcos de integração, ferramentas recomendadas e etapas para elaboração do artigo.
